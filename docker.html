☝🏻 STEP ONE : Docker

도커(Docker)는 컨테이너 기술을 사용하여 애플리케이션과 그에 필요한 라이브러리, 소프트웨어 등을 하나의 패키지로 만들어 배포할 수 있도록 하는 플랫폼이에요.

도커를 사용하면 애플리케이션을 개발할 때 필요한 환경을 포함하여 모든 종속성을 하나의 컨테이너로 만들어 관리할 수 있어요. 이렇게 만든 컨테이너는 다른 시스템에서도 동일한 환경에서 실행될 수 있어, 개발과 배포 과정을 단순화하고 효율적으로 관리할 수 있게 돼요.

위 내용이 어려울 수 있어요. 저도 처음에 “도커의 개념을 정확하게 이해해야지!” 이런 생각으로 공부하지 않고 “도커의 느낌만 이해해보자.!” 라는 생각으로 공부했어요. 솔직히 지금도 도커를 깊게 이해하고 있지 않아요. 여러분들이 공부하면 저보다 더 잘 하실 수 있어요. 위 내용이 이해 가셨다면 바로 STEP TWO : Docker Basic Command 로 넘어가시고, 어려우시다면 제가 이해한 도커를 설명해드릴게요.

도커는 “언제 어디서든 컨테이너 농사” 에요. 농사는 환경에 영향을 많이 받잖아요. 비도 와야 하고, 빛도 받아야 하고, 물도 줘야 하고요. 이외에도 태풍이 오면 열심히 농사 지은 것들이 망가지지 않도록 신경 써야 해요. 하지만 컨테이너 농사를 하면 걱정 없어요. 컨테이너는 정말 튼튼하거든요. 따라서 컨테이너에 농사를 짓고, 각 컨테이너 마다 날씨, 물 주는 주기, 거름 주는 주기를 설정하면 수성, 토성, 목성, 지구 등 환경의 영향을 받지 않고 농사를 잘 지을 수 있어요.

제가 컨테이너 농사를 좋아하는 가장 큰 이유는 사진을 찍어 현재를 저장할 수 있다는 거에요. 외부의 영향을 받지 않기 때문에 외부의 영향으로 컨테이너 내부가 손상 될 수 없어요. 하지만, 농부가 컨테이너 내부 날씨를 조절하다가 농부의 실수로 내부 농작물이 망가질 수 있어요. 이럴 때를 대비해서 농작물이 가장 잘 자랐을 때 사진을 찍어두는 거에요. 그리고 사진첩에 넣어두어야 해요. 사진을 잃어버릴 수 있으니까요. 만약, 농부의 실수로 농작물이 망가지면 사진첩에서 농작물이 가장 잘 자랐을 때 사진을 꺼내 마법으로 복원 하는 거에요. 그러면 농부의 실수는 없던 일이 돼요. 즉, 안전장치 같은 역할이에요.

컨테이너 농사의 장점을 정리해볼게요.

컨테이너는 외부의 영향을 받지 않아요. 따라서 블랙홀, 바다, 산, 목성 등 환경의 구애 받지 않고 농사 할 수 있어요. 또한, 컨테이너를 만들기 전에 컨테이너 파일을 작성해서 물을 주는 주기, 날씨 등을 파일에 작성해서 생성할 수 있어요. 더불어 농부의 실수가 아니면, 외부의 영향으로 내부가 손상되지 않아요. 또한, 농사 컨테이너를 생성할 때 더불어 현재 상황을 사진으로 찍고 사진첩에 저장만 해두면 농사를 하다가 실수해도 괜찮아요. 농작물이 전부 불타버려도 괜찮아요. 사진 첩에서 불타기 이전 상태의 사진을 꺼내서 마법으로 복원할 수 있어요. 추가적으로 사진첩이 다른 사람들과 공유만 되어 있다면, 내가 올린 사진을 다른 사람이 꺼내 복원해서 사용할 수 있어요.

이제 컨테이너 농사와 도커를 비교해볼게요.

  1. 외부의 영향을 받지 않음 = container → 저는 ubuntu 계열 사용하고 싶어요. 하지만, 제 컴퓨터의 리눅스 계열은 CentOS 에요. 이때 도커를 설치하고, ubuntu 컨테이너를 띄워서 ubuntu를 사용할 수 있어요.
  1. 컨테이너 파일 = dockerfile → 도커 파일을 작성해서 이미지를 만들 수 있어요. 도커파일에 apt-get update, apt-get install apache2 등 명령어를 작성해서 이미지를 만들 수 있어요. 이후 이미지를 컨테이너로 띄우면 설정된 명령어를 실행한 컨테이너가 생성돼요. 간단한 도커 파일을 예시를 보여 드릴게요.
FROM ubuntu:18.04                          // ubuntu:18.04 운영체제
RUN apt-get update                         // 패키지 업데이트
RUN apt-get install python3                // python3 패키지 다운로드

## 위 처럼 도커파일을 생성하고 이미지로 만들어 컨테이너를 띄우면 ubuntu18.04, 패키지 업데이트,
## python3가 설치가 진행 된 도커 컨테이너를 띄울 수 있어요.
  1. 컨테이너 사진 촬영 = docker commit → docker commit 명령어를 이용해서 현재까지 진행 된 컨테이너를 이미지로 만들 수 있어요. 이미지로 만들었다면, 작업을 하다가 알 수 없는 문제가 생겨 오류를 해결할 수 없어도 괜찮아 요. 이전 만들어진 이미지를 컨테이너로 띄우면 오류 이전 상태로 돌아갈 수 있어요.
  1. 사진첩 공유 = docker hub → 이미지를 도커 허브에 백업하여 안전하게 운영 하는거에요. 호스트에 이미지를 보관하면 실수 로 삭제할 수 있거나, 호스트에 문제가 생겨 이미지를 사용할 수 없을 수 있어요. 또한, 다른 호스트에서는 해당 이미지를 사용할 수 없어요. 따라서 도커 허브에 이미지를 백업하는 것은 안전하며, 이미지를 공유해 다른 호스트에서 사용할 수 있어요.

## 도커 버전 확인 ##
root@ubuntu:/home/study# docker -v
Docker version 20.10.21, build 20.10.21-0ubuntu1~18.04.2

## 도커 파일 관련 명령어 ##
-- 도커 파일 지시어 -- 
지시어	      설명
FROM         	베이스 이미지 지정
RUN	          이미지를 지정하면서 실행할 명령 지정
ENTRYPOINT	  컨테이너의 어플 지정 ( 컨테이너 시작할 때 실행할 명령어 )
EXPOSE	      컨테이너의 포트 지정
ADD	          이미지 생성 시 파일 추가
COPY	        이미지 생성 시 파일 복사
WORKDIR	      컨테이너 작업 디렉토리 지정
MAINTAINER	  이미지 작성자 명시
CMD	          컨테이너의 어플 지정 ( 컨테이너 시작할 때 실행할 명령어 )
LABEL	        이미지의 라벨 지정
ENV	          컨테이너의 환경 변수 지정
VOLUME	      컨테이너의 볼륨 지정
USER	        컨테이너의 사용자 지정
-- 도커 파일 생성 -- 
root@ubuntu:/home/study# docker build -t [이미지 이름] ./

## 도커 컨테이너 관련 명령어 ##
-- 도커 컨테이너 생성 --
도커 컨테이너 생성      : docker run [-옵션] [이미지 이름]
도커 컨에이너 생성 옵션 : -d -> 백그라운드 실행
                         -p [포트번호] [포트번호] -> (포트포워딩)
                         --hostname [호스트네임]  -> (리눅스 호스트 네임) 
                         --name [컨테이너 이름]   -> (컨테이너 이름)
                         -it                     -> (터미널)

🧬 도커 작업의 흐름

저는 도커를 사용하여 작업할 때 다음과 같이 작업해요. “이렇게 작업해야지!” 라고 생각했던 건 아니지만, 자연스럽게 아래와 같은 순으로 작업하고 있어요. 도움이 되실까.. 해서 정리해봐요.

1. 서버 접속

  1. 작업할 컨테이너 확인

  1. 컨테이너 진입
